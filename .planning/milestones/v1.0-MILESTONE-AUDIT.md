## Integration Findings

### MISS-01: Customer Profile Not Activated in /dsp:apply (ACTG-04)

**Severity:** Gap (blocks E2E customer overlay flow)
**File:** `.claude/commands/dsp-apply.md` line 32
**Current:** `Call load_profile(profile_name) from tools/apply_engine.py`
**Required:** `Call load_profile(profile_name, customer=overlay) from tools/apply_engine.py`

**Impact:** When running `/dsp:apply --profile engineer --overlay acme-bank`, the skill loads the base `tools/profiles/engineer.json` instead of `canon/customer/acme-bank/profiles/engineer.json`. This means acme-bank's Flink prohibition (module/flink removed) and audit role requirement (role/cp_audit added) — documented in ADR-003 — are NOT enforced through the skill.

**Evidence the feature works in isolation:**
- `apply_engine.py::load_profile("engineer", customer="acme-bank")` correctly returns the overlay profile (Phase 3C verified)
- `tests/test_profile_gating.py::TestCustomerDifferential` passes all 5 tests
- `canon/customer/acme-bank/profiles/engineer.json` exists with correct differential config

**Fix:** One-line change in `dsp-apply.md` Step 2 to pass `customer=overlay`.
