This plan deliberately copies the established CI drift-gate shape. Read these before writing new code:

**`.github/workflows/streaming-skills-drift.yml` (H.3b — closest analog)**
- Triggers: PR + push:main, path-scoped to drift-relevant files
- Setup: Python 3.12, no Node.js
- Defensive `git --version` step (fail fast if git missing)
- Invokes `python tools/check_streaming_skills_drift.py --check`
- Exit codes: 0 = no drift; 1 = drift (PR fails); 2 = config error; 3 = transient/git error

**`.github/workflows/tool-classification-drift.yml` (G.2c — earlier analog)**
- Same shape with Node.js (because that gate npm-installs mcp-confluent)
- This plan does NOT need Node.js — submodule drift uses git ls-remote only

**`tools/check_streaming_skills_drift.py` (H.3b — the script to mirror)**
- `--check` flag for CI mode
- Reads pin from `tools/vendor-sources.json`
- Compares to live `git ls-remote` upstream HEAD
- Emits clear remediation message on drift; exits 1 to block CI

The new gate differs only in:
- Pin source: the submodule's committed SHA (`git -C raw/repos/fsi-dsp rev-parse HEAD`), not a JSON file
- Comparison axis: commit timestamp delta (>14 days = stale), not SHA equality (SHAs can validly differ inside the window)

    ```python
    # Pseudocode
    def check_drift(submodule_path, upstream_remote, upstream_branch, drift_window_days, now_epoch=None):
        # 1. Resolve submodule's committed SHA (parent repo's recorded pointer)
        try:
            committed_sha = _git_rev_parse(f"HEAD:{submodule_path}")
        except subprocess.CalledProcessError:
            return (EXIT_CONFIG_ERR, f"submodule {submodule_path} not registered in parent repo")

        # 2. Fetch upstream HEAD SHA via ls-remote (no clone, no fetch)
        try:
            upstream_sha = _git_ls_remote(submodule_path, upstream_remote, upstream_branch)
        except subprocess.CalledProcessError as e:
            return (EXIT_TRANSIENT_ERR, f"git ls-remote failed: {e}")

        # 3. If SHAs match, no drift — exit OK
        if committed_sha == upstream_sha:
            return (EXIT_OK, f"OK: submodule at upstream {upstream_remote}/{upstream_branch} HEAD ({committed_sha[:12]})")

        # 4. SHAs differ — check how far behind by commit timestamp
        try:
            upstream_ts = _git_show_timestamp(submodule_path, upstream_sha)  # uses local fetch if available
        except subprocess.CalledProcessError:
            return (EXIT_TRANSIENT_ERR, f"could not resolve upstream commit timestamp for {upstream_sha}")

        now_ts = now_epoch if now_epoch is not None else int(time.time())
        delta_days = (now_ts - upstream_ts) / 86400

        if delta_days <= drift_window_days:
            return (EXIT_OK, f"OK: submodule {delta_days:.1f}d behind upstream (within {drift_window_days}d window)")

        # 5. Stale — emit remediation message and fail
        msg = (
            f"STALE: raw/repos/fsi-dsp is {delta_days:.1f} days behind "
            f"upstream {upstream_remote}/{upstream_branch} HEAD ({upstream_sha[:12]}); "
            f"drift window is {drift_window_days} days.\n"
            f"Remediate:\n"
            f"  git submodule update --remote raw/repos/fsi-dsp\n"
            f"  git add raw/repos/fsi-dsp\n"
            f'  git commit -m "chore(submodule): bump fsi-dsp to upstream main"'
        )
        return (EXIT_DRIFT, msg)
    ```

    Helper functions `_git_rev_parse`, `_git_ls_remote`, `_git_show_timestamp` each shell out via `subprocess.run([..., check=True, capture_output=True, text=True])`. Keep them small and individually monkeypatch-able from tests.

    `main()` parses `argparse` with `--check` flag, calls `check_drift()`, prints the message to stdout (exit 0) or stderr (exit nonzero), and returns the exit code.

    Then write `tests/test_check_submodule_drift.py` with 6 tests matching the behaviors above.    
    ```yaml
    name: Submodule Drift

    # Stale-submodule drift gate (Phase 9 / SUBM-03): fails the workflow when
    # raw/repos/fsi-dsp falls more than 14 days behind upstream main HEAD.
    #
    # Mirrors .github/workflows/streaming-skills-drift.yml (H.3b) shape:
    # pure shell + git ls-remote, no Node.js, no API auth.

    # Triggers scoped to changes that could affect alignment:
    #   - submodule pointer bumps (raw/repos/fsi-dsp)
    #   - the drift-check script itself
    #   - this workflow file

    on:
      pull_request:
        paths:
          - 'raw/repos/fsi-dsp'
          - 'tools/check_submodule_drift.py'
          - '.github/workflows/submodule-drift.yml'
      push:
        branches:
          - main
        paths:
          - 'raw/repos/fsi-dsp'
          - 'tools/check_submodule_drift.py'
          - '.github/workflows/submodule-drift.yml'

    jobs:
      check-drift:
        runs-on: ubuntu-latest
        steps:
          - uses: actions/checkout@v4
            with:
              # fsi-dsp is a submodule — required so the gate can resolve the
              # recorded SHA and run git ls-remote against the submodule's origin.
              submodules: true

          - name: Setup Python
            # Python 3.12 matches the other FSI CI workflows
            # (streaming-skills-drift.yml, tool-classification-drift.yml,
            # canon-parity.yml, wiki-lint.yml).
            uses: actions/setup-python@v5
            with:
              python-version: '3.12'

          - name: Verify git is on PATH
            # Fail fast with a clear message if git is missing — the gate uses
            # `git ls-remote` to fetch upstream HEAD. ubuntu-latest always has
            # git, but the defensive verb-step makes failures legible.
            run: git --version

          - name: Check raw/repos/fsi-dsp drift against upstream main HEAD
            # Exit codes:
            #   0 = within drift window;
            #   1 = stale (>14 days behind) — PR fails with remediation message;
            #   2 = config error (submodule not registered);
            #   3 = transient/git error (ls-remote failed) — fail-closed.
            # All non-zero outcomes are CI failures because they all indicate the
            # pin cannot be safely trusted as the authoritative reference.
            run: python tools/check_submodule_drift.py --check
    ```
