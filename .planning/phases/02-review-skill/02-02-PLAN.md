Create `tools/review-to-docx.py` â€” a Python CLI tool using argparse that converts a markdown review report to a .docx file with professional formatting and a CANST-04 compliant provenance footer.

**CLI interface:**
```python
#!/usr/bin/env python3
"""Convert a /review markdown report to .docx with provenance footer (REVW-03)."""
import argparse
import datetime
import re
import sys
from pathlib import Path

# Add project root to path for canon.stack import
PROJECT_ROOT = Path(__file__).resolve().parent.parent
sys.path.insert(0, str(PROJECT_ROOT))
```

Arguments:
- `input_path` (positional): Path to the .md review report
- `--floor-model` (optional, default `"sonnet"`): Floor model used during review
- `--operator` (optional, default `"claude-code"`): Operator name for provenance
- `--mcp-versions` (optional, default `"confluent-docs:latest,context7:latest"`): Comma-separated key:value pairs

**Core functions:**

1. `get_manifest_version() -> str`:
   - Read `raw/repos/fsi-dsp/MANIFEST.yaml` via `PROJECT_ROOT / "raw" / "repos" / "fsi-dsp" / "MANIFEST.yaml"`
   - Parse with `yaml.safe_load()`, return `data.get("version", "unknown")`
   - Return `"unknown"` if file not found

2. `build_provenance(floor_model: str, mcp_versions: dict, operator: str) -> dict`:
   - Import `from canon.stack import resolve_stack, active_layers`
   - Call `resolve_stack()` to get `(config, stack_hash)`
   - Call `active_layers()` to get layer names
   - Call `get_manifest_version()` for MANIFEST version
   - Return dict with keys: `stack_hash`, `canon_layers`, `manifest_version`, `floor_model`, `mcp_versions`, `timestamp` (ISO-8601 UTC), `operator`

3. `parse_markdown_report(md_text: str) -> dict`:
   - Parse the markdown report into sections using `## ` heading splits
   - Extract metadata from the header (Date, Source, Scope, Claims extracted)
   - For each `## ` section, capture the heading and body text
   - For sections containing markdown tables (lines with `|`), parse into list of row dicts
   - Return structure: `{"title": str, "metadata": dict, "sections": [{"heading": str, "body": str, "tables": list}]}`

4. `add_provenance_footer(doc: Document, provenance: dict) -> None`:
   - Add a horizontal rule paragraph (thin grey line or just spacing)
   - Build footer string: `"Canon: {canon_layers} | Hash: {stack_hash} | MANIFEST: {manifest_version} | Floor: {floor_model} | MCP: {mcp_str} | Generated: {timestamp} | Operator: {operator}"`
   - Where `mcp_str` is comma-separated `key:value` from `provenance["mcp_versions"]`
   - Add as paragraph with font size `Pt(8)`, color `RGBColor(0x88, 0x88, 0x88)`

5. `build_review_docx(md_path: Path, provenance: dict) -> Path`:
   - Create `Document()`, set Normal style: Calibri, Pt(11)
   - Parse markdown with `parse_markdown_report()`
   - Render title as Heading 0 (centered)
   - Render metadata fields (Date, Source, etc.) as normal paragraphs
   - For each section:
     - Render `## heading` as Heading 1
     - Render `### subheading` as Heading 2
     - Render body text as Normal paragraphs
     - Render tables using `doc.add_table()`:
       - First row is header (bold)
       - Set column widths explicitly to prevent overflow:
         - For Claim Validation tables (5 cols): # = 0.4in, Claim = 2.5in, Wiki = 1.2in, MCP = 1.2in, Verdict = 1.2in
         - For Premise Challenge tables (5 cols): # = 0.4in, Premise = 1.5in, Assumption = 1.5in, Challenge = 2.0in, Severity = 1.1in
         - For Canon Compliance tables (3 cols): Area = 1.5in, Status = 1.5in, Notes = 3.5in
       - Use `set_col_width()` helper (see below)
       - Table style: "Table Grid" for borders
   - Add page break before "## Recommendations" section
   - Call `add_provenance_footer(doc, provenance)` as the final element
   - Save to `md_path.with_suffix(".docx")`
   - Return the output path

6. Helper: `set_col_width(cell, width_inches: float)`:
   ```python
   from docx.oxml.ns import qn
   from docx.oxml import OxmlElement

   def set_col_width(cell, width_inches):
       tc = cell._tc
       tcPr = tc.get_or_add_tcPr()
       tcW = OxmlElement('w:tcW')
       tcW.set(qn('w:w'), str(int(width_inches * 1440)))
       tcW.set(qn('w:type'), 'dxa')
       tcPr.append(tcW)
   ```

**Main block:**
```python
if __name__ == "__main__":
    parser = argparse.ArgumentParser(description="Convert review .md to .docx with provenance")
    parser.add_argument("input_path", type=Path, help="Path to review .md report")
    parser.add_argument("--floor-model", default="sonnet", help="Floor model (default: sonnet)")
    parser.add_argument("--operator", default="claude-code", help="Operator name (default: claude-code)")
    parser.add_argument("--mcp-versions", default="confluent-docs:latest,context7:latest",
                        help="Comma-separated key:value MCP versions")
    args = parser.parse_args()

    if not args.input_path.exists():
        print(f"Error: {args.input_path} not found", file=sys.stderr)
        sys.exit(1)

    mcp_dict = dict(kv.split(":") for kv in args.mcp_versions.split(","))
    provenance = build_provenance(args.floor_model, mcp_dict, args.operator)
    out = build_review_docx(args.input_path, provenance)
    print(f"Generated: {out}")
```

# ADR-001: Producer Compression â€” zstd over lz4

## Status

Accepted

## Context

Acme Bank's Kafka clusters are storage-constrained due to regulatory retention requirements (7-year minimum for transaction events). The base canon recommends lz4 for throughput, but Acme Bank's workloads are write-heavy with long retention and benefit more from higher compression ratios.

## Decision

Override `producer.compression_type` from `lz4` to `zstd` for all Acme Bank producer configurations. The ~15% throughput reduction is acceptable given the ~30% storage savings on retained data.

## Consequences

- All producer configs reviewed under this overlay will flag lz4 as a deviation
- Performance-sensitive producers (market data ingestion) may need individual exceptions documented as engagement-level overrides
- Monitoring must track compression CPU overhead â€” alert if producer CPU exceeds 70% sustained
```

# ADR-002: Tightened Latency SLA Tiers

## Status

Accepted

## Context

Acme Bank's market data platform feeds algorithmic trading systems with sub-100-microsecond requirements, stricter than the FSI base tier of sub-millisecond. Their risk calculation pipeline also requires <5ms end-to-end, tighter than the FSI standard of <10ms.

## Decision

Override FSI latency tiers for Acme Bank:
- `market_data`: `sub-100-microsecond` (was `sub-millisecond`)
- `risk`: `<5ms` (was `<10ms`)
- `compliance` and `reconciliation` tiers remain unchanged

## Consequences

- Documents claiming sub-millisecond is sufficient for market data will be flagged as deviating under this overlay
- Architecture reviews must consider co-located brokers or dedicated clusters for market data topics
- Risk pipeline reviews will flag any design with measured p99 > 5ms
```
